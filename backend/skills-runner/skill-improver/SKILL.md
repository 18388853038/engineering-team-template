# ⚡ Skill Improver — 技能自我优化

## 概述

周期性扫描团队所有已安装的技能，从完整性、准确性、时效性和复用价值四个维度进行评估，输出优化建议并执行改进。这是 Hermes Agent "技能随使用而优化"核心机制的 OpenClaw 实现。

## 触发时机

| 触发方式 | 说明 |
|---------|------|
| **每周审查** | 每周定期对全部技能做一次健康检查 |
| **新技能安装后** | 安装新技能后检查是否与现有技能有重叠 |
| **CEO 指令** | 老板或小龙主动要求技能审查 |
| **重大任务后** | 完成大型任务后，检查相关技能是否需要更新 |

## 审查维度

### 1. 完整性检查

| 检查项 | 标准 |
|--------|------|
| 有 SKILL.md | 技能必须包含完整的 SKILL.md 文件 |
| 有 name/description | frontmatter 必填字段齐全 |
| 步骤完整 | 工作流步骤可独立执行，不依赖上下文 |
| 有检查清单 | 输出质量可量化验证 |

### 2. 准确性检查

| 检查项 | 标准 |
|--------|------|
| 描述与实际一致 | SKILL.md 描述的内容与实际能力匹配 |
| 步骤可执行 | 每一步都有明确的执行指令 |
| 依赖正确 | 引用的工具/命令在当前环境可用 |

### 3. 时效性检查

| 检查项 | 标准 |
|--------|------|
| 不过时的技术 | 推荐的技术栈/工具是否还是当前最佳实践 |
| 不引用已废弃的版本 | 命令、API 版本是否有效 |
| 更新日志 | 技能是否有版本记录 |

### 4. 复用价值

| 检查项 | 标准 |
|--------|------|
| 使用频率 | 这个技能被调用了多少次？低频则考虑归档 |
| 独特性 | 是否与另一个技能高度重叠？重叠则考虑合并 |
| 价值密度 | 这个技能解决的是真问题还是伪问题？ |

## 输出格式

```markdown
# 📊 技能健康审查报告

审查日期: YYYY-MM-DD
审查人: Skill Improver
技能总数: N

## ✅ 健康技能
- [skill-name] — 完整、准确、活跃使用中

## ⚠️ 需优化的技能
- [skill-name] — 问题描述 + 建议改进

## 🗑️ 建议归档/合并
- [skill-A] + [skill-B] — 重叠度 XX%，建议合并

## 💡 建议新增技能
- 基于近期任务模式识别，建议创建 [skill-name]
```

## 工作流程

### 定期审查流程

```
技能目录扫描
    ↓
逐一加载 SKILL.md
    ↓
4 维度评分（每项 0-5 分）
    ↓
生成失分项列表
    ↓
输出优化报告
    ↓
等待 CEO 确认后执行修改
```

### 按需优化流程

```
CEO: "看看 xxx 技能还能不能优化"
    ↓
定位该技能的 SKILL.md
    ↓
分析实际使用记录 + 当前最佳实践
    ↓
输出优化建议
    ↓
CEO 确认 → 修改文件
```

## 注意事项

- 优化前先备份原文件（创建一个 `.bak` 副本）
- 优化后运行 `openclaw config validate` 确保配置无误
- 不要删除或修改其他开发者创建的技能，除非确认未在使用
- 技能内容要通用，不要包含具体项目的硬编码信息
