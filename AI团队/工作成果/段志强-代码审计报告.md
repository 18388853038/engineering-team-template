## 二、发现的安全缺陷

### [P0/严重]

#### 1. CEO工具路径遍历漏洞 (server-modern.js:181, 191)

**问题描述：**
CEO 工具的 `read_file` 和 `write_file` 使用 `path.resolve(BASE, args.filepath)` 解析路径。在 Node.js 中，`path.resolve` 在遇到 **绝对路径** 参数时（如 `/etc/passwd`、`C:\Windows\System32\drivers\etc\hosts`），会直接返回该绝对路径，忽略 BASE 参数：

```javascript
// read_file 路径遍历 (line 181)
var content = fs.readFileSync(path.resolve(BASE, args.filepath), 'utf-8');

// write_file 路径遍历 (line 191)
var fp = path.resolve(BASE, args.filepath);
fs.writeFileSync(fp, args.content || '', 'utf-8');
```

**利用难度：** 中等。攻击者需要能触发 CEO 工具链（通过 `/api/chat/ai_ceo` 端点发送诱导指令让 CEO 调用 `read_file`/`write_file`，或直接逆向解析工具调用参数）。

**修复方案：**
```javascript
case 'read_file': {
  try {
    var resolved = path.resolve(BASE, args.filepath);
    if (!resolved.startsWith(BASE)) throw new Error('路径越权');
    var content = fs.readFileSync(resolved, 'utf-8');
    // ...
```

#### 2. 动态字段名 SQL 注入 (database.js:195)

**问题描述：**
`taskOps.update` 方法将对象键名直接拼接进 SQL 语句的 SET 子句，未对字段名进行任何校验或转义：

```javascript
update: (id, fields) => {
    fields.updated_at = new Date().toISOString();
    const keys = Object.keys(fields);
    const sets = keys.map(k => `${k} = ?`).join(', ');
    // SQL: UPDATE tasks SET ${sets} WHERE id = ?
    db.prepare(`UPDATE tasks SET ${sets} WHERE id = ?`).run(...vals, id);
},
```

**利用场景：** 攻击者发送 `POST /api/tasks/{id}` 带有形如 `{"status = 'admin'--": "x"}` 的字段时，生成的 SQL 为：
```sql
UPDATE tasks SET title = ?, status = 'admin'-- = ? WHERE id = ?
```
`--` 注释掉后续条件，实现未授权状态篡改。

**利用难度：** 中等偏高，需要枚举可用的列名。

**修复方案：** 白名单字段名或使用列名列表而非动态键名：
```javascript
const ALLOWED_FIELDS = new Set(['title', 'description', 'status', 'priority', 'assignee_id', ...]);
update: (id, fields) => {
    const safeFields = {};
    for (const [k, v] of Object.entries(fields)) {
        if (ALLOWED_FIELDS.has(k)) safeFields[k] = v;
    }
    // ...
}
```

### [P1/高危]

#### 3. 零认证机制 — 全部 API 端点完全开放 (server-modern.js: 全篇)

**问题描述：**
从 CORS 策略（`Access-Control-Allow-Origin: *`）到所有路由处理器，整个 HTTP 服务没有任何形式的身份认证、会话令牌、API Key 校验或 IP 白名单。攻击者只要能连接到 127.0.0.1:8003（或在同一局域网/反向代理暴露时）即可完全控制系统。

**影响范围：**
- 任意读写任务、团队成员数据
- 调用 AI 对话接口消耗 API 配额
- 触发沙箱代码执行（sandbox/execute）
- 读写任意工作区文件
- 发送渠道消息（微信/飞书/钉钉/企微）

**利用难度：** 低。任何能发起 HTTP 请求的进程/页面均可操作。

**修复方案：**
```javascript
// 1. 添加 API Key 中间件
const API_KEY = process.env.ECOMPANY_API_KEY || '';
function auth(req, res) {
  if (API_KEY && req.headers['x-api-key'] !== API_KEY) {
    json(res, { error: 'unauthorized' }, 401);
    return false;
  }
  return true;
}
// 2. 绑定到 127.0.0.1 仅本机访问
// 3. 如需跨网络访问，使用 TLS + 强认证
```

#### 4. API Key 明文存储与 Process 环境变量泄露 (server-modern.js + ai-engine.js)

**问题描述：**
- **文件存储：** API Key 以 JSON 明文存储在 `ai-provider.json` 中（server-modern.js:640）
- **环境变量传播：** 两处代码将 API Key 写入 `process.env`（ai-engine.js:84; server-modern.js:612），`process.env` 可通过子进程继承、崩溃 dump、`/proc/self/environ`（Linux）泄露
- **内存暴露：** `getActiveApiKey()` 和 `getAIProvider()` 将明文 Key 返回给调用方

**利用难度：** 低。任何可以读取文件系统或查看进程信息的攻击者即可获取。

**修复方案：**
```javascript
// 1. 使用 OS 级别密钥链（如 keytar）
// 2. 或系统环境变量直接注入（避免二次写入文件）
// 3. 文件存储时至少加密：
function encryptApiKey(key) {
  const cipher = crypto.createCipher('aes-256-gcm', MASTER_KEY);
  // ...
}
// 4. 运行时使用 WeakRef 或闭包而非 process.env
```

#### 5. 无请求体大小限制 — 内存耗尽 DoS (server-modern.js:413-417)

**问题描述：**
`parseBody` 函数直接拼接整个请求体到内存，无任何大小上限：

```javascript
function parseBody(req) {
  return new Promise(resolve => {
    let body = '';
    req.on('data', chunk => body += chunk);  // ← 无限制
    req.on('end', () => { try { resolve(JSON.parse(body)); } catch(e) { resolve({}); } });
  });
}
```

**利用难度：** 极低。发送一个带有超大 body 的 POST 请求即可耗尽 Node.js 进程内存。

**修复方案：**
```javascript
const MAX_BODY = 2 * 1024 * 1024; // 2MB
function parseBody(req) {
  return new Promise((resolve, reject) => {
    let body = '';
    let size = 0;
    req.on('data', chunk => {
      size += chunk.length;
      if (size > MAX_BODY) { req.destroy(); reject(new Error('Request body too large')); }
      body += chunk;
    });
    req.on('end', () => { try { resolve(JSON.parse(body)); } catch(e) { resolve({}); } });
  });
}
```

#### 6. Math.random() 生成 UUID — 可预测性 (server-modern.js:408, v4-dispatch.js:29)

**问题描述：**
UUID 使用 `Math.random()` 而非 `crypto.randomUUID()` 生成。`Math.random()` 在 V8 中是 xorshift128+ 伪随机数生成器，攻击者可通过观测若干 UUID 序列反向预测后续值。

```javascript
function uuid() {
  return 'xxxxxxxx-xxxx-4xxx-yxxx-xxxxxxxxxxxx'.replace(/[xy]/g, c => {
    const r = Math.random() * 16 | 0, v = c === 'x' ? r : (r & 0x3 | 0x8);
    return v.toString(16);
  });
}
```

**利用难度：** 中。需要获取足够的 UUID 样本（通常几百个）才能发起预测攻击。

**修复方案：**
```javascript
const crypto = require('crypto');
function uuid() { return crypto.randomUUID(); }
```

#### 7. API Key/Token 在 URL 查询参数中传递 (channels.js:35, 139)

**问题描述：**
微信 access token 和企业微信 webhook key 直接拼接在 URL 中传递：

```javascript
// personal_wx (line 35)
'https://api.weixin.qq.com/cgi-bin/message/custom/send?access_token=' + accessToken

// wecom webhook (line 139)
'https://qyapi.weixin.qq.com/cgi-bin/webhook/send?key=' + key
```

**风险：** URL 会被记录到服务器访问日志、代理日志、浏览器历史、HTTPS 解析日志等，导致凭据泄露。

**利用难度：** 中等。需要访问日志存储或 HTTP 请求日志。

**修复方案：** 使用 Authorization header 传递（但受限于微信/企微 API 设计，此为外部约束。建议限制环境内对日志的访问权限，并增加日志脱敏处理）。

### [P2/中危]

#### 8. CORS 全开放 (server-modern.js:471-473)

**问题描述：**
```javascript
res.setHeader('Access-Control-Allow-Origin', '*');
```
允许任意域名的网页跨域访问 API。结合#3（无认证），意味着任何第三方网站可以在用户浏览器中发起 API 请求。

**利用场景：** CSRF 攻击。恶意网站通过 `<script>` 或 `fetch` 调用 `http://127.0.0.1:8003/api/chat/ai_ceo` 消耗 API 额度或发起操作。

**修复方案：** 绑定 localhost 时限制 Origin 为自身，或移除 CORS header。

#### 9. 沙箱代码执行无限制 (server-modern.js:452-457)

**问题描述：**
```javascript
registerRoute(['POST'], /^\/api\/sandbox\/execute$/, async (req, res) => {
  const body = await parseBody(req);
  if (!body.code) { json(res, { error: '缺少 code' }, 400); return; }
  try {
    const result = await processSandbox.execute(body.code, body.language || 'js', { timeout: body.timeout || 30000 });
    json(res, { result, sandboxed: true });
  } catch(e) { json(res, { error: e.message }, 500); }
});
```
允许通过 POST 请求执行任意代码（JS 语言）。即使有沙箱，仍可能被绕过实现逃逸（`child_process`、`fs` 等），且执行模式、资源限制未明确定义。

#### 10. 错误信息泄露敏感路径/配置 (多处)

**问题描述：**
多处直接将原始错误对象/消息返回给客户端：

```javascript
// server-modern.js 各类接口:
catch(e) { json(res, { error: e.message }, 500); }
// 可能泄露：文件路径、SQL错误、API 响应内容
```

`ceo_error.log` 和 `ceo_debug.log` 也记录了详细的内部错误信息，包括 API 响应文本。

#### 11. API Key 掩码绕过逻辑 (server-modern.js:627-637)

**问题描述：**
```javascript
if (body.apiKey.indexOf('***') !== -1 || body.apiKey.indexOf('_BACKEND_KEY_') !== -1) {
    // ... 保留后端已有真实 key
}
```
攻击者如果知道密钥格式（含 `_BACKEND_KEY_`），可以通过发送 "掩码格式" 的 Key 来跳过更新操作。不过被跳过后仍保留原有 Key，不算直接泄露，但绕过逻辑不严谨。

#### 12. Feishu Token 缓存无并发保护 (channels.js:47-57)

**问题描述：**
```javascript
let feishuTokenCache;
let feishuTokenExpiry;
async function getFeishuToken(appId, appSecret) {
  if (feishuTokenCache && Date.now() < feishuTokenExpiry) return feishuTokenCache;
  // ...
  feishuTokenCache = data.tenant_access_token;
  feishuTokenExpiry = Date.now() + (data.expire || 7200) * 1000 - 60000;
}
```
高并发场景下多个请求同时发现缓存过期，会同时请求飞书 API 获取 token，导致请求放大和潜在限流。

#### 13. 渠道消息日志包含敏感数据 (channels.js:10-13)

```javascript
function log(channel, event, data) {
  const entry = JSON.stringify({ timestamp: new Date().toISOString(), channel, event, ...data });
  fs.appendFileSync(path.join(LOG_DIR, 'channels.jsonl'), entry + '\n', 'utf-8');
}
```
日志内容可能包含消息原文、配置参数等敏感信息。`channels.jsonl` 文件无任何权限保护。

### [P3/低危]

#### 14. 系统提示词拼接未转义 (server-modern.js:641, 656)

Agent 名称和描述直接拼接进 LLM 系统提示：
```javascript
'你是 ' + agent.name_cn + '，担任 ' + agent.title + '。' + (agent.description || '')
```
如果 `agent.json` 中的名称/描述被篡改，可实现提示注入。但该数据源为本地文件，需要先有其他入侵途径。

#### 15. 无速率限制

所有 API 端点均无请求频率限制，可被用于暴力枚举、API 配额耗尽攻击。

#### 16. 内部通信无认证 (openclaw-bridge.js)

eCompany → OpenClaw Gateway 的 HTTP 调用均无认证机制：
```javascript
const res = await fetch(`http://127.0.0.1:${GATEWAY_PORT}/api/message`, ...)
```
如 Gateway 暴露到网络，可被直接利用。

#### 17. 进程 env 变量污染 (ai-engine.js:84)

```javascript
if (raw.apiKey) process.env[PROVIDERS[raw.provider]?.apiKeyEnv || 'DEEPSEEK_API_KEY'] = raw.apiKey;
```
动态设置 `process.env` 可能覆盖已有变量或导致子进程继承 Key。Node.js child_process 默认继承父进程环境变量。
